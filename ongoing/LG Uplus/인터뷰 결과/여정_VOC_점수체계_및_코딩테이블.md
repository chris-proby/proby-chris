# 여정 맵용 VOC → 점수 체계 (인터뷰 기반)

> **상태:** `40. 신사업팀 (바이브코딩)/져니맵/index.html`에서 `VOC_CODING_FIRST_PASS` → `aggregateScoresFromVocTable()` → `JOURNEY.lgu` / `JOURNEY.skt`의 `score`(0~1)에 **반영됨** (페이지 로드 시 `applyScoresFromVocCoding()` 실행). KT는 본 테이블에 전용 VOC가 없어 HTML의 placeholder(0.45) 유지.

---

## 1. 목적

인터뷰 전사에서 **고객 여정 10단계**별로 긍·부정이 드러나는 **표현(단어·문장)**을 뽑고, **강도**를 추정한 뒤 **1~10점**으로 부여한다. 이후 단계에서 브랜드×단계별로 **집계 규칙**(평균, 가중치 등)을 정해 `0~1` 스케일로 환산할 수 있다.

---

## 2. 여정 10단계 정의 (여정 맵 `step-bar`와 동일)

| 코드 | 단계명 | 코딩 시 포함 경험 예시 |
|------|--------|------------------------|
| S01 | 전환 동기 | 이탈·유치 계기, 보안/단말/가격 등 전환 이유 |
| S02 | 정보 탐색 | 카페·유튜브·커뮤니티, 품질·가격 정보 탐색 |
| S03 | 통신사 선택 | 번호이동·지원금·채널(온·오프) 선택 |
| S04 | 개통 방식 | 개통 절차, 대리점/온라인 안내, 유심·의무기간 설명 등 |
| S05 | 배송·언박싱 | 배송 속도, 수령, 박스/구성품 첫인상 |
| S06 | 온보딩 | 개통 직후~초기 문자·앱·설정·혜택 첫 안내 |
| S07 | 서비스 이용 | 멤버십·혜택 실사용, 요금제 체감 |
| S08 | 네트워크 품질 | 속도·끊김·혼잡 지역 체감 |
| S09 | CS 대응 | 콜센터·민원·처리 결과 |
| S10 | 추천 의향 | NPS 유사 점수, 재선택·추천 발언 |

한 발화가 여러 단계에 걸치면 **주된 해석 단계 하나**에만 배치하거나, 필요 시 **행을 분리**한다.

## 3. 점수 체계 (요청안)

| 구분 | 점수 구간 | 해석 |
|------|-----------|------|
| 매우 높은 긍정 | **8 ~ 10** | 강한 만족·강한 추천·“엄청”, “매우”, 구체적 호평 |
| 일반 긍정 | **5 ~ 7.99** | 무난한 만족, 일부 긍정, “괜찮다”, “도움 됐다” |
| 일반 부정 | **2 ~ 4.99** | 불편·아쉬움·기대 이하, “딱히 없다”, “도움 안 됐다” |
| 매우 부정 | **1 ~ 2** | 강한 실망·분노·이탈 동기 수준, “아예 없다”, 반복 강조, 척도상 최악 |

**경계:** `2`점이 “매우 부정”과 “일반 부정”에 겹침. 운영 시 둘 중 하나로만 쓰도록 **단일 기준**을 정하는 것을 권장한다.

- **권장 A:** 매우 부정 **1.0 ~ 2.0**, 일반 부정 **2.01 ~ 4.99**
- **권장 B:** 매우 부정 **1 ~ 2 미만**, 일반 부정 **2 ~ 5 미만** (2는 일반 부정의 하단)

본 테이블에서는 **권장 A**에 맞춰, 극단적 발화만 2.0 이하로 둔다.

## 4. 강도 추정 로직 (가이드)

코더(또는 추후 LLM)는 각 **근거 문장**에 대해 아래를 순서대로 판단한다.

1. **극성:** 긍정 / 부정 / 혼합 → 혼합이면 **우세한 쪽** + 보조 메모.
2. **강도 신호 (언어)**
   - **상:** `엄청`, `너무`, `정말`, `완전`, `아예`, `전혀`, `딱히 … 없다`, 동일 불만 **반복**, 설문 **4번(분노/포기)** 등
   - **중:** `좀`, `조금`, `그냥`, `무난`, `괜찮`, `딱히` (무난·중립 쪽)
   - **하:** 사실 서술만 있고 평가 언어가 약함 → 보통 **5~6** 근처 중립에 가깝게
3. **행위적 결과:** 이탈·전환·추천 거절·재이동 의사 → 부정/긍정 강도 **상향**.
4. **대상 명시:** 어떤 **브랜드·채널**(LG U+ / SKT / 공통)에 대한 평가인지 태깅.

---

## 5. 브랜드×단계 집계 (다음 단계에서 결정)

인터뷰 1명에서 같은 (브랜드, 단계)에 VOC가 여러 개일 수 있다.

- 예: **단순 평균**, **최소값 보수 반영**, **대표 VOC 1개만 채택**, **가중치(이탈 직전 경험 가중)** 등.
- 여정 맵의 `0~1` 축: **적용 공식** `normalized = (단계별 평균 점수 − 1) / 9` (1점→0, 10점→1).
- **데이터 없음:** 해당 (브랜드, 단계)에 VOC가 한 건도 없으면 **중립 5.0**을 평균으로 취해 동일 공식 적용.

## 7. 여정 맵 반영 규칙 (구현)

| 항목 | 내용 |
|------|------|
| 소스 코드 | `VOC_CODING_FIRST_PASS` (V01~V31, `id` · `brands` · `step` · `s`) |
| 브랜드 태깅 | `대상 맥락`이 **LG U+ 유치·이탈·과거·LG U+** → `lgu`만. **SKT 신규·과거** → `skt`만. **단말(공통)·단말+망 인지** → `lgu`와 `skt` **동일 점수로 이중 태깅**. |
| 단계 | V12는 표기 `S04→S05`이나 **개통 맥락**으로 **S04(개통 방식)**에만 집계. |
| 집계 | 브랜드×단계별 **산술평균** → `(평균 − 1) / 9` (소수 넷째 자리 반올림). |
