원본: `아이폰 고객 인터뷰_가이드라인_초안_0319_0.5 (2).xlsx` 시트 `가이드라인_0319`

---

## 1. JSON 필드 역할 (프로비 `probing_plan`)

| 필드 | 역할 | 이 인터뷰에서의 해석 |
|------|------|----------------------|
| **`root_note`** | 이 시드 질문을 둘러싼 모더레이터 맥락·주의·엑셀 배너 문구 | 예: 「이전 vs 현재 통신사 비교」, 「엑셀 Q9」 |
| **`root_checklist`** | **이전(동일 블록 내) 답변을 바탕으로 달성해야 할 목표**. 목표가 채워지지 않으면 **추가 질문을 이어가도 된다**는 실행 지시 | 인사이트·행동 단위로 쪼갬 (예: NPS 점수·이유·개선 제안이 한 줄로 연결될 때까지) |
| **`branches`** | **답변 조건에 따른 로직 분기**. “A라고 하면 B를 더 묻는다 / C면 D로 간다” | 엑셀에 명시된 **가입 채널·결정 순서·온라인 전용 블록** 등 |

`root_checklist`는 “필수로 던지는 문장”이 아니라, **시드 질문 한 번으로 부족할 때 어떤 목표를 채우기 위해 프로빙할지**를 적어 둔 것이다.

## 3. 분기 트리(엑셀에서 직접 도출한 `branch` 후보)

### 3.1 통신사 vs 단말기 **결정 순서** (엑셀 Q9, 안내 행 15)

- **조건**: 참여자가 **통신사를 먼저** 정했다.
  - **진행**: `[통신사 인지/탐색]` 묶음(Q12~Q16)을 **먼저** 깊게 본 뒤, `[단말기 결정]`(Q17~)으로 넘어간다.
- **조건**: **아이폰(단말)**을 먼저 정했다.
  - **진행**: `[단말기 결정]`(Q17~)을 **먼저** 깊게 본 뒤, `[통신사 인지/탐색]`으로 넘어간다.
- **조건**: 둘을 동시에/반복하며 정했다.
  - **진행**: 실제로 **시간 순서대로** 재구성해 질문 순서를 맞춘다.

*(엑셀 15행: “통신사/단말기 결정 순서에 맞게 [통신사 인지/탐색]과 [단말기 결정]을 질문”)*

### 3.2 **가입 채널** — 온라인 vs 오프라인 (엑셀 Q25~, Q29 배너)

- **조건**: **온라인**으로 가입·개통 절차를 밟았다.
  - **진행**: **배송/수령·지연·안내(Q29~Q36)** 블록을 심층한다.
- **조건**: **오프라인(매장 등)** 위주였다.
  - **진행**: 배송 블록은 생략·축소하고, **매장 방문·수령·상담** 경험으로 대체 질문한다.

*(엑셀 Q29 배너: “(온라인 채널 가입 고객만) 단말기 배송/수령”)*

### 3.3 **NPS** (엑셀 Q6·Q8)

- **공통**: 점수를 말한 뒤 **이유**와 **점수를 올리려면 무엇이 필요한지**까지 한 흐름으로 묻는다 (체크리스트로 고정).

### 3.4 **이전 vs 현재 통신사** (배너 11·58행)

- **조건**: 구간 B·D 전반이 이 규칙을 공유한다.
  - **진행**: 동일 질문이라도 **이전 통신사 대비**로 답하도록 유도한다 (`root_note`).

### 3.5 **익시오/에이닷** (엑셀 Q80~Q82)

- **조건**: **둘 다** 써 봤다.
  - **진행**: 차이·선호(Q82)를 반드시 묻는다.
- **조건**: **하나만** 써 봤다.
  - **진행**: 비교 질문은 생략하고 경험·불만/긍정 한 가지씩만 확보한다.

## 4. `root_checklist` 설계 원칙 (이 파일 전반)

1. **한 줄 = 한 목표**: “무엇이 말로 확보되면 이 시드에서 넘어가도 되는가”를 적는다.  
2. **추가 질문 허용**: 목표가 비어 있으면 시드만으로 끝내지 않고 **추가 질문**해도 된다는 뜻으로 읽힌다.  
3. **엑셀의 `-` 보조 문항**은 “꼭 던질 하위 문장”이 아니라, **체크리스트에 흡수**해 목표로 옮긴다 (예: 채널 예시 → “정보 채널이 구체적으로 밝혀질 때까지”).  
4. **비고(F열)**는 `root_note` 또는 첫 번째 체크리스트(운영·캡처 요청 등)로 반영한다.

## 6. JSON `branches` 객체 형식 (이번 생성물)

각 분기는 아래 키를 가진 객체로 넣었다.

| 키 | 필수 | 설명 |
|----|------|------|
| `id` | ✓ | UUID 문자열 |
| `position` | ✓ | 배열 내 순서 (0부터) |
| `when` | ✓ | **조건**: 어떤 답·상황일 때 이 분기를 택하는지 |
| `then` | ✓ | **그때**: 추가 질문·블록 순서·진행 지시 |
| `else` | | **그렇지 않을 때**: 대체 진행 (한 조건에서 양자 택일일 때만 사용) |

예: Q25(가입 채널)는 `when`이 온라인 가입일 때 `then`으로 배송 블록 심층, `else`로 오프라인 대체를 둔다.
