# `/sk-1` — 전체 프로파일 생성

## 개요

`self.csv` + `metadata.csv` 두 파일을 읽어 유저별 프로파일 `.md` 파일과 코드 정의 `metadata.md`를 자동 생성하는 스킬.

## 사용법

```
/sk-1 [디렉토리 경로]
```

경로를 생략하면 현재 작업 디렉토리에서 실행됩니다.

## 입력 파일

| 파일 | 설명 |
|------|------|
| `self.csv` | 유저별 자기소개서 데이터 (1행 = 1유저) |
| `metadata.csv` | 코드 정의표 (전문분야 코드, 성공경험 수준, 에세이 질문, 세부항목 유형) |

## 출력 파일

| 파일 | 설명 |
|------|------|
| `metadata.md` | 코드 정의 레퍼런스 문서 |
| `profiles/{이름}_{GUID}.md` | 유저별 프로파일 (1인 1파일) |

## 처리 파이프라인

### 1. metadata.csv 파싱

- **전문분야 코드맵** — `L/M/S` 코드 계층 구조 파싱
  - L코드 (상위): Business/Marketing, Production, R&D, Corporate Staff 등
  - M코드 (중위): Legal, Finance, HR 등
  - S코드 (하위): Domestic Legal Affairs, Global Finance 등
- **성공경험 수준 메타** — `성공경험수준X_Y` → 역량 영역 레이블 + 10점 척도 기준
- **세부항목 유형 코드** — `성공경험 세부항목체크X_Y` → 성공경험번호별 A/B/C/D 의미 매핑
- **에세이 문항** — `ESSAY_Q1~Q6` → 역량명 + 실제 질문 텍스트

### 2. self.csv 파싱 및 프로파일 생성

각 유저 행에 대해:

#### 2-1. 기본 정보
GUID, 소속, 조직, 직책

#### 2-2. 전문분야 (1~3순위)
- 전문분야 코드 → 명칭으로 치환
- 상위 카테고리(Corporate Staff 등) 병기
- 경험 연수, 업무내역, 대표 업적 포함

#### 2-3. 성공경험 수준 (테이블)
- 12개 역량 영역 × 점수 × 수준 레이블
- 10점 척도: 1=경험없음/보조적 → 3=실무 수행 → 5=핵심 주도 → 7=전사적 성과 창출 → 9=산업/시장 변혁 주도

#### 2-4. 세부 경험 (중복 제거 후 통합)

**중복 감지 기준:**
1. 제목 유사도 ≥ 0.55 (SequenceMatcher)
2. 키워드 겹침 ≥ 2개 AND 비율 ≥ 0.50
3. "앞서…같음" 패턴 → 참조 항목으로 처리

**통합 규칙:**
- 그룹 내 내용이 가장 풍부한 항목을 대표로 선택
- 참조 항목 제거, 활용된 성공경험 번호 병기
- 유형 코드(A/B/C/D) → 해당 성공경험의 실제 의미로 치환

#### 2-5. 대표 성공경험

#### 2-6. 에세이 (Q1~Q6)
- 컬럼명(ESSAY_Q1) → 실제 질문 텍스트로 치환
- 역량명 병기

### 3. metadata.md 생성

| 섹션 | 내용 |
|------|------|
| 성공경험 수준 | 컬럼명 / 역량 영역 / 10점 척도 기준 테이블 |
| 세부항목 유형 코드 | 성공경험 1~5별 A/B/C/D 의미 테이블 |
| 에세이 문항 | 컬럼명 / 역량 / 질문 테이블 |
| 전문분야 코드 | L→M→S 계층형 테이블 |

## 코드 치환 규칙 요약

| 원본 | 치환 결과 | 기준 |
|------|-----------|------|
| `M0030` | Legal (Corporate Staff) | code_map + parent_map |
| `L0004` | Corporate Staff | code_map |
| `A^B` (성공경험1) | 기존 사업 성장 / 신사업 개척 | check_meta[1] |
| `D` (성공경험1) | 유의미한 경험 없음 | check_meta[1] |
| `ESSAY_Q1` | Q1. 지금의 경험과 역량을… | essay_meta |
| `7점` | 전사적 성과 창출 경험 | score_label() |

## 파일 명명 규칙

```
profiles/{성명}_{GUID}.md
```

예: `김윤욱_E690072.md`, `이종현_E660011.md`

# `/sk-2` — 2차 요약 파일 생성

## 개요

`/sk-1`이 생성한 `profiles/` 폴더와 원본 `self.csv` + `metadata.csv`를 읽어,
각 유저별로 **대표 성공경험 + 에세이 6문항**만 추린 2차 `.md` 파일을 생성하는 스킬.

## 사용법

```
/sk-2 [디렉토리 경로]
```

경로를 생략하면 현재 대화 맥락의 디렉토리를 사용합니다.

## 입력 파일

| 파일 | 설명 |
|------|------|
| `self.csv` | 유저별 원본 데이터 (`대표성공경험` 컬럼 포함) |
| `metadata.csv` | 에세이 질문 및 세부항목 유형 코드 정의 |

## 출력 파일

| 파일 | 설명 |
|------|------|
| `2차/{이름}_{GUID}.md` | 유저별 2차 요약 파일 |

## 처리 파이프라인

### 1. metadata.csv에서 파싱

- `essay_meta` — `ESSAY_Q1~Q6` → (역량명, 실제 질문 텍스트)
- `check_meta` — 성공경험 번호별 A/B/C/D → 의미 텍스트
- `level_meta` — `성공경험수준X_Y` → 역량 영역 레이블

### 2. self.csv에서 유저별 처리

#### 2-1. 대표 성공경험 매칭

`대표성공경험` 컬럼값(제목)을 기준으로, 성공경험 1~5의 세부경험 전체에서 **유사도 매칭**:

- SequenceMatcher 유사도 ≥ 0.5인 항목 중 가장 높은 것 선택
- 매칭된 항목의 내용이 참조("앞서…같음")인 경우, 같은 제목의 비참조 항목으로 대체
- `대표성공경험` 미지정 시: *대표 성공경험이 지정되지 않았습니다.* 표시

#### 2-2. 대표 성공경험 내용 출력

- 경험 제목 (h3)
- 해당 성공경험 번호의 수준 점수 (역량 영역 + 점수)
- 유형: A/B/C/D → `check_meta[best_exp_num]` 기준으로 치환
- 전체 경험 내용

#### 2-3. 에세이 출력 (Q1~Q6)

- `essay_meta`에서 실제 질문 텍스트로 헤더 치환
- 역량명 이탤릭 표시
- 답변 내용

### 3. 출력 파일 구조

```
# {이름} — 대표 성공경험 & 에세이

**소속:** ...  **조직:** ...  **직책:** ...

---

## 대표 성공경험

### {경험 제목}
**성공경험 N 수준:** {역량영역}: {점수}점 / ...
**유형:** {A/B/C/D 치환 결과}
{경험 내용}

---

## 에세이

### {Q1 실제 질문}
*역량: {역량명}*
{답변}

... (Q2~Q6 반복)
```

## 파일 명명 규칙

```
2차/{성명}_{GUID}.md
```

# `/sk-3` — 6frame 분류 분석 파일 생성

## 개요

`2차/` 폴더의 유저별 요약 파일과 `6frame.md` 평가체계를 읽어,
각 유저별로 **6frame L3 항목별 가장 부합하는 발췌문**을 매핑한 3차 분석 파일을 생성하는 스킬.

Python 스크립트 없이 LLM 기반 내용 분석으로 수행합니다.

## 사용법

```
/sk-3 [디렉토리 경로]
```

경로를 생략하면 현재 대화 맥락의 디렉토리를 사용합니다.

## 입력 파일

| 파일 | 설명 |
|------|------|
| `6frame.md` | 6frame 평가체계 레퍼런스 (L1·L2·L3 구조 및 각 항목별 관찰 포인트) |
| `2차/{이름}_{GUID}.md` | 유저별 대표 성공경험 + 에세이 요약 파일 |

## 출력 파일

| 파일 | 설명 |
|------|------|
| `3차/{이름}_{GUID}.md` | 유저별 6frame 분류 분석 파일 |

## 6frame 구조 (3축)

| L1 | L2 | L3 항목 |
|----|----|---------|
| 경험 | 전문성 | Biz. 도메인, Function 전문성 |
| 역량 | Design 역량 | 문제 구조화, 종합적 기획력, 변화/혁신 |
| 역량 | Learning Agility | 미래 준비 (Sense of Urgency), Globality, AX 혁신 |
| 자질 | Attitude | Humility, Integrity, Resilience |
| 자질 | Attitude ⚠️ | 나르시즘, 미시적 관리, 과도한 야망, 타인조작, 공격성, 수동공격성, 냉담성 |
| 자질 | Leadership | 통찰과 전략 (지), 실행과 혁신 (용), 포용과 통합 (덕) |

## 처리 파이프라인

### 1. 6frame.md 숙지

- L1·L2·L3 계층 구조 파악
- 각 L3 항목의 관찰 포인트 이해

### 2. 2차 파일 읽기

- 대표 성공경험 전문 읽기
- 에세이 Q1~Q6 전문 읽기
- 대표 성공경험 미지정 여부 확인 (미지정 시 Q1 내용으로 대체)

### 3. L3 항목별 발췌 매핑 (LLM 분석)

각 L3 항목에 대해:
1. 해당 항목의 관찰 포인트가 드러나는 문장/단락을 전체 답변에서 탐색
2. 가장 부합하는 2~4문장을 발췌 (직접 인용)
3. 발췌 출처를 *(대표 성공경험)*, *(Q1)*, *(Q2~Q6)* 등으로 명시
4. 해당 내용이 없는 경우 `*(해당 언급 없음)*` 기재

### 4. 다크사이드 항목 처리 (Attitude ⚠️)

- 직접 언급이 없어도 **간접 신호** 포착:
  - Q5 "없음" 또는 Q6 "잘 모르겠음" → 나르시즘 간접 신호로 기록
  - 자기 리스크에 대한 성찰 회피 응답 → 자기인식 결여 신호
  - 외부 피드백 언급 (부정적 평가 수용 여부) → 나르시즘·공격성·과도한야망 신호
- 간접 신호 포함 시 발췌 뒤에 *(신호 근거 설명)* 추가

### 5. 출력 파일 작성

## 출력 파일 구조
