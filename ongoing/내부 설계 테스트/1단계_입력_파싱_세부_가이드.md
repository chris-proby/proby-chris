# 1단계 입력 파싱 — 세부 설계 가이드

> 적용 대상: `%내부평가` 단축어. 사용자 입력(1.~6. 양식)을 파싱해 JSON 객체로 만드는 규칙.

---

## 1. 파싱 대상 식별

- **트리거:** 메시지에 **`%내부평가`** 가 포함된 경우.
- **파싱 구간:** `%내부평가` 뒤부터 메시지 끝까지.  
  - 다음 블록 시작은 **`숫자.`** (예: `1.`, `2.`) 로 구분.
- **여러 블록:** 연속된 **1. ~ 6.** 세트가 반복되면 **블록이 여러 개**로 간주.  
  - 예: `1. 질문A` … `6. 결과A` 다음에 `1. 질문B` … `6. 결과B` 가 오면 **2개 블록** → JSON에 **객체 2개** 추가.

---

## 2. 항목별 매핑 및 세부 규칙

### 2.1 `1.` → `질문` (문자열)

| 항목 | 설명 |
|------|------|
| **JSON 키** | `질문` |
| **타입** | 문자열 (string) |
| **추출** | `1.` 다음 줄의 내용. 같은 줄에 있으면 `1. ` 뒤의 문자열만 추출. |
| **빈 경우** | 내용이 없으면 `""` |
| **예시** | `1. 주 거래 은행은 어디인가요?` → `"주 거래 은행은 어디인가요?"` |
| **비고** | 시드 질문 한 개. 줄바꿈 포함 시 첫 줄만 사용하거나, 한 문장으로 합쳐 저장. |

---

### 2.2 `2.` → `노트` (문자열)

| 항목 | 설명 |
|------|------|
| **JSON 키** | `노트` |
| **타입** | 문자열 (string) |
| **추출** | `2.` 다음부터 **`3.`** 이 나오기 전까지의 모든 줄. 여러 줄이면 공백/줄바꿈을 한 칸으로 합치거나 그대로 유지. |
| **빈 경우** | 내용 없음 → `""` |
| **권장 형식** | `조건: ... 액션: ...` (조건부 지시일 때) |
| **예시** | `2. 조건: 참여자가 신한은행을 주 거래 은행으로 언급한 경우. 액션: 해당 블록 안에서 브랜드 컬러 관련 질문 1회 포함` → 한 문자열로 저장. |
| **비고** | 조건·액션만 담고, 가이드 문장(선호 이유 등)은 3·5번으로. |

---

### 2.3 `3.` → `체크리스트` (문자열 배열)

| 항목 | 설명 |
|------|------|
| **JSON 키** | `체크리스트` |
| **타입** | 문자열 배열 (array of strings) |
| **추출** | `3.` 다음부터 **`4.`** 이 나오기 전까지.  
  - **한 줄 = 한 요소:** 빈 줄이 아닌 각 줄을 배열 요소 하나로.  
  - 또는 문장 단위로 나눠서(마침표/줄바꿈 기준) 한 요소로. |
| **빈 경우** | 내용 없음 → `[]` |
| **예시** | 3번 구간에 "주 거래 은행 선호 이유를 최소 1회 질문으로 탐구" / "신한은행 언급 시 브랜드 컬러 질문 1회 포함" 두 줄 → `["주 거래 은행 선호 이유를 최소 1회 질문으로 탐구", "신한은행 언급 시 브랜드 컬러 질문 1회 포함"]` |
| **비고** | "한 행동 = 한 요소" 원칙. 실행 가능한 할 일만 포함. |

---

### 2.4 `4.` → `규칙` (문자열 배열)

| 항목 | 설명 |
|------|------|
| **JSON 키** | `규칙` |
| **타입** | 문자열 배열 (array of strings) |
| **추출** | `4.` 다음부터 **`5.`** 이 나오기 전까지.  
  - 한 줄 = 한 규칙 요소. 또는 문장 단위로 분리. |
| **빈 경우** | 내용 없음 → `[]` |
| **예시** | 4번 구간에 "동일·유사 질문 반복 금지" 한 줄 → `["동일·유사 질문 반복 금지"]` |
| **비고** | 반복 금지, 횟수 제한, 돌발 대응 등 제약 문장. |

---

### 2.5 `5.` → `프로빙가이드` (문자열 배열)

| 항목 | 설명 |
|------|------|
| **JSON 키** | `프로빙가이드` |
| **타입** | 문자열 배열 (array of strings). **한 줄이어도 배열로 통일.** |
| **추출** | `5.` 다음부터 **`6.`** 이 나오기 전까지.  
  - 한 줄 = 한 목표(Desired insight) 요소. |
| **빈 경우** | 내용 없음 → `[]` |
| **예시** | 5번 구간에 "선호 이유 1개 이상 도출" / "참여자 발화 인용·확인 후 연결" 두 줄 → `["선호 이유 1개 이상 도출", "참여자 발화 인용·확인 후 연결"]` |
| **비고** | "이만큼 이해되면 블록 완료"인 목표. 인용·표현 다양화 원칙도 여기 포함 가능. |

---

### 2.6 `6.` → `결과` (문자열 배열)

| 항목 | 설명 |
|------|------|
| **JSON 키** | `결과` |
| **타입** | 문자열 배열 (array of strings). 순서 = 대화 턴 순서. |
| **추출** | `6.` 다음부터 **다음 블록 `1.`** 또는 메시지 끝까지.  
  - **방식 A:** **`- `** (하이픈+공백) 로 시작하는 각 줄 = 대화 턴 한 개.  
  - **방식 B:** `- ` 없이 줄마다 턴으로 처리(빈 줄 제외).  
  - 두 방식 모두 지원 시, `- ` 가 있으면 A, 없으면 B 적용. |
| **빈 경우** | 내용 없음 → `[]` |
| **따옴표** | 참여자 발화에 큰따옴표가 있으면 JSON에서 `\"` 로 이스케이프. |
| **예시** | `- 주 거래 은행은 어디인가요?` / `- 신한은행입니다.` → `["주 거래 은행은 어디인가요?", "신한은행입니다."]` |
| **비고** | 질문/답변 구분 없이 턴 순서만 유지. 홀수번=모더레이터 등 규칙은 별도 정의. |

---

## 3. 숫자·구분자 인식

- **패턴:** `1.`, `2.`, … `6.`  
  - **인식:** `숫자` + `.` 또는 `숫자` + `. ` (점 뒤 공백 허용).
- **다음 블록 시작:** 새로 등장하는 `1.` 은 다음 블록의 시작.  
  - 이전 블록의 `6.` 구간은 그 `1.` 직전까지.

---

## 4. 빈 항목 처리

- **문자열 필드** (질문, 노트): 내용 없음 → `""`
- **배열 필드** (체크리스트, 규칙, 프로빙가이드, 결과): 내용 없음 → `[]`
- **공백만 있는 줄:** 빈 항목으로 간주하거나, 배열에서는 요소로 넣지 않음.

---

## 5. 최종 JSON 객체 형태

한 블록 파싱 결과는 반드시 아래 키를 가진 객체 하나.

```json
{
  "질문": "string",
  "노트": "string",
  "체크리스트": ["string", "..."],
  "규칙": ["string", "..."],
  "프로빙가이드": ["string", "..."],
  "결과": ["string", "..."]
}
```

- **프로빙가이드**는 기존 `bank_interview_qa.json`과 맞추기 위해 **항상 배열**로 저장. (한 줄이어도 `["한 줄"]`)

---

## 6. 요약 표

| 입력 | JSON 키 | 타입 | 빈 값 |
|------|---------|------|--------|
| 1. | 질문 | string | `""` |
| 2. | 노트 | string | `""` |
| 3. | 체크리스트 | string[] | `[]` |
| 4. | 규칙 | string[] | `[]` |
| 5. | 프로빙가이드 | string[] | `[]` |
| 6. | 결과 | string[] | `[]` |
